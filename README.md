# Исследование возможностей чат-ботов с ИИ в добывающей промышленности
## Автор: (Литвяков Максим Тарасович]
## Цель исследования
Сравнить качество решения прикладных задач горнорудной отрасли с помощью
зарубежного (DeepSeek) и отечественного (GigaChat) ИИ-ассистентов.
## Используемые модели
1.
**Зарубежный бот:** [DeepSeek]
2. **Отечественный бот:** [GigaChat]
## Выбранные кейсы
1. **Кейс 1: Производственная безопасность**
* *Описание: * Моделирование ситуации с возгоранием ГСМ на карьере.
* *Цель: * Проверить способность ботов к техническому анализу и следованию логике.
## Методология
1. Сформулирован единый запрос для каждого кейса.
2. Запросы отправлены обоим ботам в одном временном контексте (новый чат).
3. Результаты сохранены в папку 'results/.
4.Проведен сравнительный анализ по критериям:
полнота ответа, COOTBEe
отраслевой специфике, наличие "галлюцинаций" (ложных фактов), удобство восприятия.
## Основные выводы
* *Зарубежный бот * лучше справился с задачей [действия сотрудника при обнаружении инцидента].
*Отечественный бот* продемонстрировал лучшее понимание локальных
нормативных актов, но уступил в креативности при решении задачи.
*DeepSeek: Три варианта: общий, для ГСМ, краткий по рации. Указано направление эвакуации (наветренная сторона).
*GigaChat: Один короткий текст без деталей («покинуть территорию»). Не сказано, куда именно идти.
## Как запустить/проверить
Все диалоги сохранены в виде скриншотов или текстовых логов в директории 'results/'.
Запросы находятся в папке 'prompts/".
